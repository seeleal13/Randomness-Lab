🔍 Understanding the Differences

Each type of randomness has unique characteristics and use cases. Compare their patterns and behaviors!

Pseudo-Random (LCG)

Pros: Fast, reproducible, memory efficient

Cons: Predictable with known seed, eventual repetition

Uses: Games, simulations, testing

True Random (Crypto)

Pros: Unpredictable, cryptographically secure

Cons: Slower, requires system entropy

Uses: Security, cryptography, passwords

Physical Random

Pros: User-controlled, transparent process

Cons: Requires interaction, variable quality

Uses: Interactive demos, educational purposes

Advanced RNG Comparison

Explore the trade-offs between speed, quality, and historical significance in these advanced random number generators!

Xorshift Algorithm

Pros: Extremely fast, simple implementation, good statistical properties

Cons: Can have correlations, not cryptographically secure

Uses: Gaming, Monte Carlo simulations, embedded systems

Period: 2³² - 1 (very long)

Middle Square Method

Pros: Simple concept, historically significant, easy to understand

Cons: Poor statistical properties, short cycles, can hit zero

Uses: Educational purposes, historical interest

Period: Very short (often < 100)

Multiplicative Congruential

Pros: Faster than full LCG, good for specific applications

Cons: Shorter period than LCG, can't generate zero

Uses: Older systems, simple applications

Period: 2³⁰ - 1 (good but limited)

🎯 When to Use Each Generator

🚀 Need Speed?

Use Xorshift for maximum performance in games or simulations where speed is critical.

📚 Learning?

Try Middle Square to understand RNG fundamentals and see how cycles form.

🔧 Simple Systems?

Consider Multiplicative Congruential for basic applications with limited resources.

About This Project

Welcome to the Randomness Lab an interactive educational platform I created to explore the fascinating world of random number generation, I hope you vote for my project!

Why I Built This

the idea came from my experience with a fraud ban, since i used a random.randint to generate delay between keystrokes but it was flagged because standard randomness is not truly random, that's what really made me search about this topic more, and I was really suprised when I discovered that a computer can never generate true random numbers!

I realized that most people encounter randomness every day, in games, security systems, simulations, and more. but rarely understand the mechanics behind it. This project aims to make randomness seem clear by making these complex algorithms interactive and visual.

What You'll Discover

Through hands-on experimentation, you can:

• Compare different approaches: From simple pseudo-random algorithms to cryptographically secure generators

• Visualize patterns: See how different algorithms produce unique distribution patterns

• Understand trade-offs: Learn why speed, security, and predictability matter in different contexts

• Explore history: From von Neumann's pioneering work to modern cryptographic standards

Educational Goals

This project serves multiple educational purposes:

For Students: A practical way to understand algorithms, mathematics, and computer science concepts

For Developers: Insight into when to use different types of random number generators

For Curious Minds: A small glimpse into how computers create the illusion of randomness

Technical Implementation

Built with vanilla HTML, CSS, and JavaScript to ensure maximum accessibility. No external dependencies means you can run this anywhere, anytime. The visualizations are created using HTML5 Canvas for smooth, real-time chart updates.
